0x1 服务描述

官网地址：https://www.elastic.co/

0x2 服务基础信息

elasticsearch

默认端口：9200

服务信息：ssl/http

版本信息：Elasticsearch REST API 7.0 or later (Shield plugin; realm: security)

kibaba

默认端口：5601

服务信息：http

版本信息：Elasticsearch Kibana (serverName: kibana)

Title：Elastic Kibana

0x3 服务部署

部署环境：AlmaLinux release 9.2

filebeat

下载安装包

https://www.elastic.co/cn/downloads/beats/filebeat

wget https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-8.15.2-linux-x86_64.tar.gz

解压到当前目录

tar -zxf filebeat-8.15.2-linux-x86_64.tar.gz

cd filebeat-8.15.2-linux-x86_64

修改配置文件filebeat.yml

配置数据输入
查看支持的模块

./filebeat modules list

编辑filebeat.yml
配置数据输出到elasticsearch
编辑filebeat.yml

启动测试

sudo ./filebeat -e -c filebeat.yml

后台启动

nohup ./filebeat -c filebeat.yml &

nohup ./filebeat -c filebeat.yml > /dev/null &

Elasticsearch

下载安装包

https://www.elastic.co/cn/downloads/elasticsearch

wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-8.15.2-linux-x86_64.tar.gz

或

wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-6.5.3.tar.gz

验证安装包合法性

解压到当前目录

tar -xzf elasticsearch-8.15.2-linux-x86_64.tar.gz

创建单独账号

基于安全性要求，不要使用root账号启动服务，需要单独创建服务账号

1、useradd 创建账号

2、修改elasticsearch目录归属者为新创建账号

chown -R newuser:newuser elasticsearch目录

启动测试

./elasticsearch-8.15.2/bin/elasticsearch

https://10.185.16.237:9200/

默认会配置

查看启动进程

ps -ef | grep elasticsearch

netstat -anlp | grep 9100

Kibana

下载安装包

https://artifacts.elastic.co/downloads/kibana/kibana-6.3.1-linux-x86_64.tar.gz

解压到当前目录

tar -zxvf kibana-6.3.1-linux-x86_64.tar.gz

启动测试

cd kibana-6.3.1-linux-x86_64/

./bin/kibana

备注：

有的默认会监听localhost，如果需要监听其他，

需要修改config/kibana.yml，server.host 修改为其他IP

或者使用 -H 参数

./bin/kibana -H 0.0.0.0

0x4 报错信息处理

can not run elasticsearch as root

不能用root用户启动elasticsearch，单独创建账号启动

需要把elasticsearch目录归属者设置为单独创建的账号
