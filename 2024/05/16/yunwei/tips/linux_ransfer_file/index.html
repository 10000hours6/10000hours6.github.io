linux_ransfer_file

本文最后更新于 2024-05-16T19:29:16+08:00

背景

1、需要将1台Linux服务器的数据迁移到另1台Linux服务器

2、服务器登录通过堡垒机登录，无法使用 scp 或 rsync

考虑使用 tar 和 nc 命令进行传输

命令说明

tar

unix下常用的压缩命令

本次传输所需的参数如下

-a ：archive 压缩保持权限在内的属性

-r ：recursive 递归压缩目录

nc（netcat）

本次传输所需的参数如下

-l ： 开启监听

-u：指定协议为UDP

-p：指定监听端口

需要注意的：

不同版本的nc，命令不同，nc -lp port 和 nc -l port 哪个可用就用哪个

具体操作

接收端

命令

nc -l -p 9999 | tar xf -

解释

监听端口 9999

接受到文件后解压到本地（因为是以数据流的形式），如果不解压，只会传输不会保存

文件接受完成后会自动断开连接，结束监听

示例

发送端

命令

tar cf - 目录名 | nc serverip 9999

tar cf - rcracki_mt_0.7.0_linux_x86_64.zip | nc 10.235.151.11 9999

解释

将需要传输的目录压缩转换为字节流
