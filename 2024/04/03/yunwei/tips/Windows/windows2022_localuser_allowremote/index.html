windows2022_localuser_allowremote

本文最后更新于 2024-04-03T20:29:58+08:00

背景

在Windows Server 2022 Standard 系统中新增本地用户后无法远程登录

远程报错信息

要远程登录，你需要具有通过远程桌面服务进行登录的权限。默认情况下，远程桌面用户组的成员有这项权限。如果你所属的组没有这项权限，或者远程桌面用户组中已经删除了这项权限，那么需要手动为你授予这一权限

报错原因

本地用户默认没有远程登录的权限，需要单独设置权限，管理员用户默认拥有远程登录权限

解决方案

“本地用户和组”增加权限

1、Win+R 输入 “lusrmgr.msc”进入用户和组配置界面

2、组->Remote Desktop Users 右击选择“属性”，在成员中增加对应的用户名

备注：

该权限增加后远程登录还是会提示同样的错误，需要进行下一步配置

“本地组策略”增加权限

1、Win+R 输入 “gpedit.msc”进入本地组策略编辑器界面

2、计算机配置->Windows设置->安全设置->本地策略->用户权限分配->允许通过远程桌面服务登录 右击选择“属性”，添加用户或组，添加对应的用户即可

备注：

默认只有Administrators

如果没有上一步，直接执行这一步，会出现“拒绝请求的会话访问”

完成以上两步操作后，即可远程登录
